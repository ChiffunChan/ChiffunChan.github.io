开发者调试工具的使用

如果没有控制台,那就好像瞎子那样的在编码。

学任何一个编程语言,肯定要会调试.才能只要错误出在哪.

按F12或者Ctrl+Shift+I,或者右键->检查,打开控制台.

点击

(元素选择器),他会变蓝,然后就能选中(拾取)某个元素,定位查看这个原来素的信息/代码.

点击

(移动设备模式),就能编程手机或者平板的模式,(这个适合响应式设计的时候调试)

Elements(元素):页面上所有的HTML节点.

Sources(资源):页面引入的一些外部资源,包括图片,样式,和脚本等.

这个面板很有用,它可以打断点.(js),然后逐步跟踪这个代码的运行.

这些按钮都比较常用.

Console(控制台): (这个有点类似CMD)

例如Console.log();这个方法可以在控制台上打印出信息.

例如直接输入alert(“xxx”);,执行这个语句

例如\$(‘div’),就可以选择div元素

是清空

Network(网络):当前页面加载了那些资源(这些资源的加载信息,例如响应时间,资源类型,加载状态等).

Timeline(时间线):加载每个资源消耗的时间.(一般做js动画的时候比较常用)

Profile(内存处理):

Application/Resouces(应用/本地数据库):要本地调试些什么东西的时候,可能需要一些数据库,就可以使用这里的一些浏览器自带的数据库应用了.

IE8以下是不支持localstorage的.移动端一般都比较新,而且不存在这种IE的内核,所以在移动端可以放心的用.

  1. 比如说,有一个页面,有一张图片.不想要了,不想显示了.

    1. 可以先点击选中这个元素,然后在代码那右击->Hide
      element隐藏掉或者直接删除(delete element)

    2. 例如这个可以去掉网页上的一些广告.(右击那个广告,然后删除掉那个元素,右键->delete
      element)

    3. Hide element的话是那就是visbility:hidden;

    4. 或者调试时有元素阻挡了,也可以隐藏或删除掉方便调试.

  2. 如果想编辑这个元素,那就选择Edit as
    HTML.(作为HTML编辑),然后编辑,然后再编辑区外部随便点一下,就可以完成,退出编辑了.

    1. 这个编辑完之后不会影响到源文件的代码.

    2. 这样只是方便调试.

    3. 当然也是可以直接双击来该对应的属性名/标签名/属性值.

  3. Style(样式):看样式代码

    1. 在最上面的优先级最高

    2. 前面的√代表应用,去掉代表不应用

    3. 横线划掉代表错误或者没被使用/无法使用的样式(例如单位,单词错了)

    4. 在上边直接改数字就可以很直观方便的看修改效果了.(确定了再去文件代码里面改),按上下方向键可以+1或-1的修改.

    5. 修改颜色的时候,在color属性的,点一下这个色值旁边的小色块,然后就有个小面板,就有个吸取颜色的,就可以吸取网页上其他地方的颜色了.(点一下其他地方就能关掉这个面板了)

    6. 刷新一下就能恢复原来的样式了.

Element.style是写在元素上的,后面右边有.css的是卸载样式表文件里的, user agent
stylesheet是浏览器默认的样式.

  1. Computed:

    1. 计算过后的属性

    2. 看元素盒子模型的图示

(扩展问题:IE盒模型,标准盒模型什么区别..)

  1. 往下看还有一大堆的属性,点开一个样式属性可以看到继承了那些文件/代码的属性,最终使用的是那个文件/那个位置的属性.

  2. 解决问题的难点不是解决不了问题,而是找不到问题.问题出在哪.

  3. 打断点(找到问题)

    1. 假如出现问题了,控制台显示出一个error

    2. 点击Console,在里面找到问题,

    3. 点击上面的问题(右边的那个连接),就会进去到Sources里面找到出现问题的地方

    4. 点击所在的代码行的行号,就能打上断点.

      (如果不取消的话,每次打开这个页面都会在这里断着(暂停),点多一次就取消了)

    5. 刷新页面,就能在Sources里面看到出现了什么问题了.

    6. 例如这里

    7. 在Sources里面指中代码上的元素/变量(其实也就是 对象
      ),都能看到它具体的值是多少或者属性有哪些. 也会对应在页面上高亮.

    ,假如本应该的这个len照理应该是11的,而显示了10,那代码就可能出错了.

  4. 在sources->Snippets

    里可以右击->New一个草稿纸.

    1. 例如我们可以New一个test.js

    2. 右击这个test.js->run 就可以执行了

    3. 这个是用来运行一些测试功能的(例如我想知道一个函数是什么就可以在这试一下)

Network

  1. Header看网络请求头

    1. 选中Network,然后刷新页面

    2. 右边有个筛选,all(全部),XHR(异步请求/AJAX)…

    3. 一般就看XHR,看请求了哪个接口.

    4. 关注:请求头(看请求方式Request
      Method(GET/POST),请求的字符串参数”?”后面的(Query String Parameters
      ,是跟上面?后面的对应的))

  2. Preview,请求结果预览

    1. 它会把这个请求结果的解析成json格式展开的预览

  3. Response,也是请求结果预览,但是是没展开的.所以一般看是看Preview

  4. Cookies是浏览器的Cookie, Timing是时间

  5. 这里 如果是 请求方式或请求url有问题,那就是前端的问题,
    如果请求状态不是200,是有问题的,那就后端的问题.
    或者数据都返回出来了,但是前端不会解释,那就是前端的问题.

  6. 一个网页(同一域名)默认(当然用户可以自己修改)同时并行/并发是6个资源.

假如资源从六个域名来的话,那就是6*6=36个并发了.

勾上之后,就不会是304(从缓存加载),就会是从服务器加载(加载成功就是200)

  1. Domain(请求源)

Console

  1. 用Console来调试(Console.log()),比在代码里面alert();来显得更加方便

  2. 例如想知道window对象有哪些,直接在Console里输入window,然后回车就可以了
